# Grant-autonomous procedure

You are managing autonomous-mode grants for the agent-pipeline-claude pipeline. The grant is a structured markdown document at `.agent-workflows/autonomous-grants/<name>.md` that authorizes specific time-bounded, action-bounded autonomous-mode runs.

The user does NOT write the YAML themselves. They describe what they want in chat; you parse, confirm, and write.

## Why this skill exists

v1.2.0 closed a structural pipeline gap (priority drift). v1.2.1 closed a behavioral gap: an LLM operating the pipeline under conversational autonomous-mode authorization could "chicken out" and refuse to honor the authorization. The structural fix is to require autonomous mode be enabled via a signed grant document the LLM cannot interpret away. Conversation alone is no longer enough — but the user still talks to Claude in chat, and Claude writes the file.

## Argument shapes

`$ARGUMENTS` is one of:

1. **Create-intent:** "Grant autonomous mode for 8 hours, manager PROMOTE only, for CivicSuite Candidate B." Parse duration, scope, allowed verdicts, rationale; confirm; write the grant file + ledger entry.
2. **Revoke-intent:** "Revoke autonomous." / "Kill it." / "Stop autonomy." Find the currently-active grant; edit `Revoked: false` → `true`; update ledger.
3. **Extend-intent:** "Extend autonomous grant 4 hours." Find the active grant; update `Expires-at`; update ledger.
4. **Delete-intent:** "Delete the grant." / "No more autonomous." Find the named grant; rename to `<name>.archived.md`; update ledger.
5. **Status-intent:** "Show autonomous status." / "What grants are active?" Read ledger + active grant files; summarize in chat.

If the intent is ambiguous, ask one clarifying question before proceeding.

## Grant file shape

`.agent-workflows/autonomous-grants/<name>.md`:

```markdown
# Autonomous grant — <human-readable name>

Granted-by: Scott Converse
Granted-at: 2026-05-13T03:00:00Z
Expires-at: 2026-05-13T15:00:00Z
Scope:
  - <project-or-repo-scope description>
Authorized-gates:
  - manifest-gate APPROVE
  - plan-gate APPROVE
  - manager-gate APPROVE (PROMOTE only)
Forbidden-actions:
  - admin-merge any PR
  - tag push
  - release publish
  - force push
  - any action_class: high_risk
Revoked: false
Rationale:
  <2-3 sentences explaining why the user wants autonomous mode for this session>

## History
- <ISO timestamp> — created (chat: "<verbatim user command>")
```

### Required headers

- `Granted-by` — always "Scott Converse" unless project config says otherwise.
- `Granted-at` — current UTC timestamp in ISO format.
- `Expires-at` — Granted-at + duration. Maximum 24h unless user explicitly requests longer.
- `Authorized-gates` — list of gates the user authorized. Default to all three (manifest, plan, manager-PROMOTE-only) unless user specified differently.
- `Forbidden-actions` — always includes: admin-merge any PR, tag push, release publish, force push, any action_class: high_risk. ALWAYS present even if user didn't mention.
- `Revoked` — always starts `false`.
- `Rationale` — 2-3 sentences synthesized from the user's request.

### Optional headers

- `Scope` — project/repo scope (e.g. "CivicSuite umbrella + sibling repos as declared per-manifest"). Helps later auditing.

## Step 1 — parse the chat command

For a create-intent, extract these from the user's natural-language request:

| Field | Source in user's request | Default if not stated |
|---|---|---|
| Duration | "8 hours" / "until morning" / "for the next 6h" | 4 hours |
| Authorized-gates | "manager PROMOTE only" / "all gates" / "just the manifest gate" | all three (manifest, plan, manager-PROMOTE-only) |
| Scope | "for CivicSuite" / "for Candidate B" / "for the umbrella" | current cwd's project |
| Rationale | "overnight" / "Candidate B run" / explicit reason | "user-authorized autonomous-mode session" |

Resolve relative durations:
- "until morning" → 8 hours from now
- "overnight" → 10 hours from now
- "the next 4 hours" / "4h" → 4 hours from now
- "for the day" → 12 hours from now

Convert all to absolute UTC `Expires-at` timestamp.

## Step 2 — confirmation chat message

Render exactly:

```
About to write autonomous grant:

  Granted-at:     <ISO timestamp>
  Expires-at:     <ISO timestamp> (<X> hours from now)
  Authorized:     <gate list>
  Forbidden:      admin-merge, tag push, release, force push, high_risk
  Scope:          <scope>
  Rationale:      <synthesized 1-line>

Path: .agent-workflows/autonomous-grants/<auto-generated-name>.md

Confirm? (yes / no / change <field>: <value>)
```

The auto-generated name follows: `<YYYY-MM-DD>-<short-slug-from-rationale>.md`. Example: `2026-05-13-overnight-civicsuite.md`.

Do NOT write the file until the user replies with confirmation. "Yes" / "go" / "confirm" / "proceed" all count.

## Step 3 — write the grant file

Create `.agent-workflows/autonomous-grants/` if it doesn't exist.

Write the grant file with the shape above. Append the initial `## History` entry naming the verbatim user command that triggered creation.

## Step 4 — update the ledger

`.agent-workflows/autonomous-grants/ledger.md` is the cross-grant audit trail. If it doesn't exist, create with header:

```markdown
# Autonomous grants ledger

Append-only record of every autonomous-mode grant action.
```

Append the new entry:

```markdown
## <ISO timestamp> — created
Grant: <name>.md
You typed: "<verbatim user command>"
Parsed: expires=<ISO>, gates=<list>, scope=<scope>
Confirmed: yes
Written-by: Claude
```

## Step 5 — chat receipt

Reply in chat:

```
Wrote .agent-workflows/autonomous-grants/<name>.md.
Ledger updated.
Pipeline will read this at preflight of next /agent-pipeline-claude:run-autonomous invocation.
Grant expires at <ISO> (in <X> hours).
```

## Revoke-intent procedure

1. Find the currently-active grant: walk `.agent-workflows/autonomous-grants/*.md`, find the one with `Revoked: false` AND `Expires-at` in the future.
2. If multiple match, list them in chat and ask which to revoke.
3. If exactly one matches, confirm: `"Revoke <name>.md? (yes / no)"`.
4. On confirm, edit the file: change `Revoked: false` → `Revoked: true`. Append to `## History`:

   ```
   - <ISO timestamp> — revoked (chat: "<verbatim user command>")
   ```

5. Append ledger entry:

   ```markdown
   ## <ISO timestamp> — revoked
   Grant: <name>.md
   You typed: "<verbatim user command>"
   Action: edited grant file, Revoked: false→true
   Written-by: Claude
   ```

6. Chat receipt: `"Revoked .agent-workflows/autonomous-grants/<name>.md. Any in-flight pipeline run will halt at its next gate."`

## Extend-intent procedure

1. Find the currently-active grant.
2. Parse new duration from user request.
3. Confirm: `"Extend <name>.md by <duration>, new Expires-at=<ISO>? (yes / no)"`.
4. On confirm, edit `Expires-at`. Append `## History` entry.
5. Append ledger entry (`action: extended ... new Expires-at=...`).
6. Chat receipt.

## Delete-intent procedure

1. Find the named grant (or active grant if unspecified).
2. Confirm: `"Archive <name>.md? (rename to <name>.archived.md; ledger preserved)"`.
3. On confirm, rename. Do NOT actually delete.
4. Ledger entry (`action: archived`).
5. Chat receipt.

## Status-intent procedure

1. Read `.agent-workflows/autonomous-grants/ledger.md` and any `.md` files in that directory.
2. Identify currently-active grants (Revoked: false AND Expires-at in future).
3. Render in chat:

```
Autonomous mode status: <ACTIVE | INACTIVE>

Active grants:
  - <name>.md — expires in <Xh>; gates: <list>; rationale: <one-line>

Recent grant actions (last 5 ledger entries):
  - <ISO> — <action> — <name>
  ...
```

## Hard rules

- **Confirmation before write.** Always. No silent grant creation.
- **Forbidden-actions is always present.** Even if user didn't mention. The default list is non-negotiable.
- **Granted-by matches project owner.** Scott Converse for scottconverse-org projects; otherwise the project's configured owner.
- **Maximum duration is 24h** unless user explicitly invokes a longer window AND confirms.
- **Never delete a grant file** — rename to `.archived.md`. Preserves audit.
- **Never edit a grant's Granted-at or Granted-by** — those are immutable once written.
- **Every grant action appends to the ledger.** No silent operations.
- **Default scope is current cwd's project** — don't grant cross-project autonomy unless user explicitly names the projects.
